<objective>
Fix the Docker networking bug where the Vite dev-server proxy hardcodes `http://localhost:8000` as its `/api` target. Under docker-compose the frontend container's `localhost` resolves to itself (not the backend container), so every `/api/*` call fails with ECONNREFUSED.

Purpose: Restore working API calls (upload, transactions, categories) when the stack runs under docker-compose, without breaking native host runs.
Output: Env-driven proxy target in `frontend/vite.config.ts` (falls back to localhost) + corrected `VITE_API_URL` in `docker-compose.yml`.
</objective>
